# 女装售卖系统 — 后台管理功能设计方案

## 背景
前台购物功能已完成，需要为管理员提供后台管理能力，包含商品管理、库存管理、订单管理。

## 设计目标
- 前后台通过顶部按钮一键切换，无需登录
- 所有数据存 localStorage，前后台共享同一数据源
- 后台功能完整：商品CRUD、库存调整、订单状态流转

## 架构方案

### 数据模型扩展

**商品 (Product)** — 已有，增加 `status: 'active' | 'inactive'` 字段控制上架/下架

**订单 (Order)** — 新增数据结构：
```js
{
  id: 时间戳,
  items: [{ name, price, qty, color, size, image }],
  total: Number,
  name: String,     // 收货人
  phone: String,    // 联系电话
  addr: String,     // 收货地址
  note: String,     // 备注
  status: 'pending' | 'paid' | 'shipped' | 'completed' | 'cancelled',
  time: String      // 下单时间
}
```

### 页面结构

```
┌─────────────────────────────────────┐
│  Header: 品牌 + 搜索 + 购物车 + [后台]│  ← 新增「后台管理」切换按钮
├─────────────────────────────────────┤
│  前台模式                             │
│  ┌─ 分类导航 ──────────────────────┐ │
│  ┌─ 商品网格 ──────────────────────┐ │
│  ┌─ 详情弹窗 / 购物车 / 结算 ──────┐ │
├─────────────────────────────────────┤
│  后台模式（#adminPanel）              │
│  ┌─ Tab导航: 商品管理 | 库存管理 | 订单管理 ─┐
│  │  商品管理: 表格列表 + 新增/编辑弹窗       │
│  │  库存管理: 表格列表 + 快速修改库存        │
│  │  订单管理: 表格列表 + 状态下拉变更        │
└─────────────────────────────────────┘
```

### 组件划分

| 组件 | 说明 |
|------|------|
| **模式切换** | Header中「🎛️ 后台管理」按钮，点击切换前台/后台 |
| **商品管理** | 表格展示所有商品（含状态），新增/编辑弹窗表单，下架/删除操作 |
| **库存管理** | 表格展示商品+库存数量，直接修改库存 |
| **订单管理** | 表格展示所有订单，下拉框修改订单状态 |

### 功能详情

#### 商品管理
- 表格列：图片、名称、分类、价格、库存、状态、操作
- 新增商品弹窗：名称、分类(下拉)、价格、原价、库存、描述、图片URL、颜色、尺码
- 编辑商品：复用新增弹窗，预填数据
- 下架：修改商品 status = 'inactive'，前台不再显示
- 删除：从数组中移除（确认弹窗）
- 上架：修改 status = 'active'

#### 库存管理
- 表格列：图片、名称、分类、当前库存、操作
- 点击「修改」弹出输入框，直接修改 stock 值

#### 订单管理
- 表格列：订单号、收货人、电话、金额、状态、时间、操作
- 状态下拉：待付款 → 已付款 → 已发货 → 已完成 / 已取消
- 点击展开查看订单详情（商品列表）

### 数据初始化
- 首次加载时，将 products 数组写入 localStorage('womenProducts')
- 订单写入 localStorage('womenOrders')
- 前台读取 localStorage 中的 products，而非硬编码数组
- 兼容已有购物车数据 localStorage('womenCart')

4. 添加前后台切换逻辑
