# 系统设置重设计 v2 — 全屏版 UI 与交互方案

_设计日期: 2026-05-24 | 状态: 原型定稿 | 原型: prototypes/prototype-settings-v2.html_

---

## 1. 整体架构

### 1.1 三层数据模型

```
Provider ─── 管理凭据 (Base URL + API Key)
    │
    ├── 模型 1 ─── 带默认参数 (temperature/max_tokens/context_length) + 能力标记 (vision/function_calling)
    ├── 模型 2
    └── 模型 3 …
    
Route ─── 选 Provider → 选 Model → 继承默认参数(可覆盖) → 可选关闭能力
```

| 层 | 职责 | 典型字段 |
|----|------|---------|
| Provider | API 凭据 + 关联模型列表 | id, name, base_url, api_key, type(openai-compatible/local) |
| Model | 模型元数据 + 默认参数 + 能力声明 | name, temperature, max_tokens, context_length, vision, function_calling |
| Route | 路由配置 = Provider + Model + 参数覆盖 | id, name, provider_id, model_name, temperature_override, max_tokens_override |

### 1.2 页面定位

- 从抽屉（v1）改为**全屏页面**（v2），sidebar 新增「⚙ 系统设置」入口
- 内容和场景广场/工具列表同级，独立占据内容区
- 不再需要半透明遮罩层，交互更直接

## 4. 🔀 路由配置

### 4.1 路由表

7 列 + 1 展开列，grid 布局：

| 列 | 宽度 | 内容 |
|----|:----:|------|
| ▶展开 | 28px | 旋转 90° 箭头 |
| 路由 | 120px | 路由名 + 图标 |
| Provider | 1fr | select 下拉框 |
| 模型 | 1fr | select 下拉框 |
| 温度 | 80px | input + "默认 X.X" |
| 最大 Token | 100px | input + "默认 XXXX" |
| 惩罚 | 80px | input + "默认 X.XX" |
| 能力 | 60px | 能力图标聚合 |

### 4.2 5 个路由定义

| 路由 | 图标 | 说明 |
|------|------|------|
| 频道闲聊 | 💬 | 默认路线，日常对话 |
| 场景分析 | 🔧 | 结构化分析场景 |
| 约束提取 | 🔗 | 提取约束条件 |
| Medium | ⚖️ | 中等负载任务 |
| Heavy | 🏋️ | 高负载/复杂任务 |

### 4.3 参数覆盖

- 参数默认跟随模型自身的默认值（灰色小字 "默认 X.X" 显示）
- 用户修改后 input 边框变蓝（border-color: #58a6ff），表示已覆盖
- input 的 font-size: 14px（与已实现页面统一）
- "默认 X.X" 使用 position: absolute 定位在 input 下方，不占文档流

### 4.4 展开行（能力覆盖）

每行路由可展开详细的能力配置：

```
┌──────────────────────────────────────────────────┐
│ ▶ 💬 频道闲聊   [DeepSeek▼] [model▼] 0.70 8192 1.05 🖼🔧 │
├──────────────────────────────────────────────────┤
│ ⚡ 能力覆盖（可选）                              │
│ [🖼 Vision（来自模型）] [🔧 Function Calling] [📤 流式输出]│
└──────────────────────────────────────────────────┘
```

- 能力来自模型的声明（"来自模型" 后缀）
- 用户可关闭某项能力（slider 开关）
- 模型不支持的能力显示为灰色不可开启

### 4.5 底部操作区

```
┌──────────────────────────────────────────────────┐
│ 🔵 蓝色边框 = 参数已覆盖 · 🖼🔧 = 能力图标 ...   │
├──────────────────────────────────────────────────┤
│  [💾 保存路由配置]            [重置]              │
└──────────────────────────────────────────────────┘
```

- legend-box 在上，按钮行在下
- 保存按钮（绿色 primary）在左，重置按钮（透明 ghost）在右
- 重置按钮样式：btn-ghost（透明背景，hover 显示文字色）

## 5. 📝 系统人设

### 5.1 交互流程

```
默认状态 → textarea readonly, 灰边框, 不可编辑
    │
    ├── 点击 [✏️ 编辑] ──→ 解除 readonly, 蓝色聚焦边框
    │                      右下角浮现 [取消] [恢复默认] [保存]
    │
    ├── [取消] ──→ 恢复 readonly, 按钮消失
    ├── [保存] ──→ 保存后恢复 readonly, 按钮消失
    └── [恢复默认] ─→ 弹出确认, 恢复默认文案, 恢复 readonly
```

### 5.2 卡片结构

```
┌──────────────────────────────────────────────────┐
│ 💬 频道人设                          [✏️ 编辑]   │
├──────────────────────────────────────────────────┤
│ ┌──────────────────────────────────────────────┐ │
│ │ 你是坐山客（Zuoshanke）...                    │ │  ← readonly textarea
│ └──────────────────────────────────────────────┘ │
│           [取消]  [恢复默认]  [保存]              │  ← 编辑时才显示
└──────────────────────────────────────────────────┘
```

两个卡片独立控制，互不影响。

## 9. 后续编码建议

### 9.1 实现顺序

```
Stage 1: Provider 模型层
  └── providers 表 + CRUD API (Base.metadata.create_all())
  
Stage 2: 路由配置改造
  └── routing schema 扩展 (provider_id 替代 provider_str)
  
Stage 3: ai_engine.py 泛化
  └── 通用 OpenAI-compatible 调用，移除硬编码分支
  
Stage 4: 前端页面实现
  ├── Provider 管理组件（卡片列表 + 编辑/新增表单）
  ├── 路由配置组件（路由表格 + 展开行 + 参数覆盖）
  ├── 系统人设组件（readonly/编辑双态 + 操作按钮）
  └── 服务状态组件（卡片网格 + 展开详情 + 刷新）
  
Stage 5: 迁移与兼容
  └── 首次启动从环境变量自动创建 Provider/Model
```

### 9.2 关键技术点

- **select/input 字体统一 14px**，与已实现页面 body font-size 一致
- **参数覆盖用蓝色边框标记**（border-color: #58a6ff），纯 CSS，无需特殊状态管理
- **能力覆盖展开行使用 toggle 模式**，与路由展开复用一套逻辑
- **服务状态卡片展开**用 `grid-column: 1 / -1` + `display: contents` 包装器，不破坏网格
- **系统人设 textarea readonly 态**用 CSS 属性选择器控制样式，无额外状态
