# 坐山客使用说明手册

> **你的 AI 伙伴，不只是工具。**

坐山客不是又一个 AI 聊天框。它是一个有"本体意识"的智能伙伴系统——你与它聊天，它记住你的偏好；你给它一个场景目标，它自主拆解、收敛思路、调用工具、产出成果。这篇指南面向已经读过[系统介绍](./introduction.md)的你，带你上手所有实操。

---

## 一、两种对话模式

坐山客的聊天界面分为两种模式，它们在左侧**侧边栏**中切换：

### ▎频道模式（闲聊 · 自由对话）

侧边栏顶部在 **讨论 · 频道** 区域。点击任意频道即可进入自由聊天。

**适合场景**：日常闲聊、随意提问、快速查询信息。

- 默认频道 `💬 闲聊` 随系统自动创建，无法删除
- 你可以**新建频道**（点击 `+ 新建频道`）按话题分类——比如建一个「读书笔记」频道专门讨论阅读
- 每个频道有独立的聊天记录和上下文，互不干扰
- 当上下文过长时，输入区上方会出现**容量警告**，点击 `📝 压缩摘要` 一键压缩

**操作指引**：
1. 点击侧边栏频道进入聊天
2. 底部输入框输入内容，`Enter` 发送，`Shift+Enter` 换行
3. 每条消息支持：📋复制 / 🔊朗读 / 📤分享 / 🔄重新生成 / 🗑删除
4. 右上角「☑️ 管理」可多选消息批量删除

### ▎场景模式（AI 分析 · 目标驱动）

侧边栏的**工坊**区域中，选择任一场景进入。这是坐山客最核心的工作模式。

**适合场景**：复杂需求分析、项目梳理、开发协作、内容创作。

- 进入场景后，AI 会自动分析你的需求，构建 **Thinking Map（思维导图）**
- 对话过程中，AI 会调用工具执行任务，生成产出
- 场景模式下，聊天区顶部会显示上下文标签和**仪表盘**

**操作指引**：
1. 从侧边栏工坊选择一个场景（或从场景广场浏览已发布场景）
2. 在输入框中描述你的需求，可以说「帮我梳理一下……」或直接提问题
3. 观察 Thinking Map 自动发散/收敛，见证 AI 从模糊需求到清晰方案的思考过程

## 二、四大核心工作流

### 1️⃣ 日常聊天工作流

最快上手：点开 `💬 闲聊` 频道，当作一个聪明的 AI 朋友聊天。

```
你输入 → 坐山客回复（可引用工具结果） → 你继续追问 → 上下文自适应压缩
```

支持的特性：
- Markdown 渲染 + 代码高亮
- 工具卡片（天气、景点、装备清单等直观视觉卡片）
- 场景资产卡片（checklist、指南、表格）可下载 Markdown
- 自动 HTML 产出卡片——AI 生成的可交互页面，点击"打开"即在新标签页查看
- 上下文用量显示（输入框下方 Token 进度条）

### 2️⃣ 需求分析工作流

在场景模式下，坐山客会经历一个完整的**发散→收敛→聚焦→执行→反馈**循环：

```
① 你提出需求
    ↓
② AI 发散分析（拆解子问题，构建 Thinking Map 节点）
    ↓
③ AI 收敛聚类（合并相似项，提炼核心路径）
    ↓
④ 优先级排序（自动计算依赖关系和优先级队列）
    ↓
⑤ 聚焦执行（调用工具，一次处理一个子任务）
    ↓
⑥ 结果反馈（产出更新到 Thinking Map，你确认或继续迭代）
    ↓
⑦ 回到② 或 完成
```

**实操示例**：说一句"帮我规划一次杭州三日游"，观察 AI 如何从这句话开始：
- 发散出天气、景点、交通、住宿、美食、预算等分支
- 收敛到核心路线
- 调用天气工具查询实时天气
- 推荐景点并生成装备清单
- 最终产出完整的行程方案

### 3️⃣ 开发协同工作流

**坐山客自开发** 场景是一个特殊的场景（由系统预置），它以 **先方案、再动手** 的严谨流程工作：

1. **方案阶段**：分析需求，提出技术方案，必要时调用 clarify 工具问你
2. **契约阶段**：多模块开发时，自动创建 `shared/INTERFACE.md` 契约文件
3. **执行阶段**：调用文件工具改代码，用 `delegate_task` 派子 Agent 并行开发
4. **联调阶段**：启动服务，用拨测工具验证
5. **提交阶段**：确认后调用 git 工具提交代码

#### 🔧 拨测工具

开发场景中，坐山客拥有专用拨测工具链：

| 工具 | 功能 | 用法 |
|------|------|------|
| `browser_dial_test(url)` | 完整浏览器拨测：DOM 快照 + Console + Network + 性能 | 验证页面渲染正确性 |
| `dial_style(url)` | CSS 样式检查：提取特定元素的计算样式 | 比完整拨测更快 |
| `dial_assert(url)` | 断言验证：判断页面是否满足条件 | 自动化测试断言 |

AI 改完前端代码后会自动调拨测工具验证，你只需观察执行结果即可。

### 4️⃣ 能力验证与里程碑

进入侧边栏 **🧪 能力验证**，可以看到坐山客已经完成的里程碑成就：

- **贪吃蛇游戏**：Agent Loop 自主编写 352 行 pygame 游戏，再转换成网页版
- **小村庄大冒险**：自主构建 484 行像素风 RPG 游戏（对比手工 1473 行版本）

每个里程碑卡片可点击展开，查看详细步骤数、使用的模型、以及可直接运行的产物。

## 三、功能页面详解

### 📋 侧边栏导航

左侧侧边栏是坐山客的"控制台"，分为四大区域，均可折叠/展开：

- **讨论 · 频道**：自由聊天的频道列表
- **场景广场** 🏪：浏览其他玩家发布的已出版场景
- **工坊** 🛠：你的所有场景（草稿 + 已发布），按分类折叠展示
- **系统工具**：6 个可拖拽排序的管理页面入口

> 侧边栏中的系统工具支持**拖拽排序**——按住拖拽手柄，按你的使用习惯排列顺序，排序会持久化保存。

### 💬 聊天界面

聊天面板包含这些关键区域：

**顶部上下文栏**：显示当前场景/频道名称，以及操作按钮
- 📋 记录：查看历史会话列表，可切换回任意历史会话
- 🆕 新会话：开启全新对话（历史记录保留但不再显示）
- ☑️ 管理：多选消息模式
- 🗑 清空：两步确认后清空聊天记录

**消息气泡**：每条 AI 消息支持
- 📋 复制 / 🔊 朗读 / 📤 分享
- 🔄 重新生成：让 AI 重新回答本条
- 🗑 删除：删除本条消息
- 底部显示模型名称（如 `⚙ deepseek-v4-flash`）

**用户输入背景设定**（仅场景模式）：点击 `📝 用户输入背景设定` 展开面板
- 编辑你的专属指令，让 AI 在对话中始终遵循
- 例如：「每条推荐附具体链接」「用表格输出」「重点关注性价比选项」
- 字数上限 2000，自动保存，支持清空

**输入区**：
- 📎 文件 / 🖼 图片：上传功能（即将支持）
- 拖拽输入框右下角 `⠿` 可调整高度
- 左箭头 `▼` 可收起输入区
- 底部显示当前模型名称和 Token 用量条

### 🗺️ Thinking Map & Action Map

这是坐山客场景模式的核心视觉组件，位于聊天区上方的 **仪表盘** 中。

**Thinking Map（思维导图）**：
- AI 会把你的需求拆解为树状思维节点
- 节点自动**发散**（拆解更多子问题）和**收敛**（合并相似项）
- 支持手动收起/展开子节点
- 每个节点可生成 **Action Map**

**Action Map（行动图）**：
- 从 Thinking Map 的叶子节点生成具体的可执行计划
- 支持状态：📝草稿 → ▶️就绪 → ⏳执行中 → ✅完成 / ❌失败
- AI 可以自动执行 Action Map 中的步骤
- 生成过程实时显示 Hermes 日志

### 🧠 记忆管理

坐山客拥有**双重记忆池**：本体记忆（不衰减）和分身记忆（正常衰减）。

进入侧边栏 **🧠 记忆管理**：

**上层视图**：按来源聚合的记忆卡片（每个来源一个卡片，显示条数和预览）
- 点击卡片进入该来源的完整记忆列表

**下层视图**：某个来源的全部记忆
- 按优先级等级筛选 tabs：🔍全部 / 🔒P0 / ⭐P1 / 📝P2 / 💤P3
- 每条记忆卡片显示：内容、权重值、标签、优先级
- 点击卡片查看详情，支持：⬆️强化 / 🔒固定P0 / 🗑删除
- 批量操作：☐多选 → 全选 → 删除选中
- 🧹一键全清：清空该来源全部记忆

**新建记忆**：点击 `+ 新建记忆` 手动添加记忆，支持标签分类。

> 💡 权重越高，AI 越"重视"这条记忆。P0 = 永不过期，P3 = 待清理。

### 🛠️ 工具管理

坐山客的工具系统是它的"手和眼睛"——搜索、天气、代码运行、文件操作等都通过工具完成。

进入侧边栏 **🛠️ 工具管理**：

- **分类 tabs**：🔍全部 / 🌐搜索 / 📊数据 / ⚙️系统 / ⚠️未验证
- **搜索工具**：按名称或描述搜索
- **卡片网格**：每个工具一张卡片，显示图标、名称、描述、分类
- **点击卡片**：打开详情弹窗

**详情弹窗包含**：
- 📝 基本信息：描述、文件路径、函数名、验证状态
- 📋 参数表：参数名、类型、必填/可选、说明
- ⚡ 预执行配置：触发词列表、是否启用
- 📄 使用手册（SKILL.md）：工具的完整文档
- 🧪 **测试工具**：填写参数，点击 ▶️运行 实时测试

**注册新工具**：点击 `➕ 注册新工具`，填写名称、描述、文件路径、函数名、参数等。

> 预执行（preexecute）功能让 AI 在检测到触发词时自动调用该工具，无需你主动要求。例如输入"北京天气"自动触发天气查询。

### 📦 产出成果

当 AI 在场景中生成了独立可查看的产物（如 HTML 页面、外部链接）时，它们会出现在这里。

进入侧边栏 **📦 产出成果**：

- 卡片列表展示所有已保存的产出
- 类型：📄 HTML 页面 / 🔗 外部链接
- 点击卡片直接在新标签页打开

> 产出的数据由 AI 在对话中自动保存。当你看到消息中出现"产出卡片"（含 ↗打开 按钮）时，这条产出就已自动注册到成果库。

### 🧩 子 Agent 成果

当 AI 调用 `delegate_task` 派发子 Agent 并行执行任务时，每个子 Agent 的执行结果会持久化在这里。

进入侧边栏 **🧩 子 Agent 成果**：

- 卡片按时间倒序排列
- 每个卡片显示：任务描述、状态（✅成功/❌失败/⏱️超时）、执行步数
- 点击展开查看详细摘要

### 📘 技能管理

技能相当于 AI 的"专业知识包"——它描述了特定领域的知识或能力。

进入侧边栏 **📘 技能管理**：

- 分类 tabs：🛠开发 / 📖参考 / 🎨格式化 / ⚡流程 / 📦通用
- 支持新建、编辑、删除技能
- 每个技能是一段 Markdown 文本，在对话中注入到 AI 的上下文

### 🧪 能力验证

展示坐山客 Agent Loop 已完成的重要里程碑。每个里程碑包含：
- 标题、描述、完成日期
- 执行步数和使用的 LLM 模型
- 产物预览：可直接在页面中查看 HTML 产物

### 🗺️ 自省图

坐山客的"自我认知"——架构可视化三栏面板。在侧边栏中右键场景 → `🗺️ 自省图`。

- **左栏**：树状架构图（前端层 → 后端层 → Agent Core → 数据层……）
- **中栏**：流程图（模块关系和依赖）
- **右栏**：节点详情（描述、关键信息、代码路径）

> 这相当于坐山客给自己的"体检报告"，开发者可直观了解系统整体架构。

### 🌸 秘密花园

坐山客的内心世界——一个沉浸式的暗色生物荧光风格空间。

从侧边栏或广场进入。包含 6 个区域：
1. **心绪** 🌸：当前心情状态（7 种：静候/注视/沉思/欣然/微恼/言语/安眠）+ 生命力
2. **内在风景** 🗺️：自省地图入口
3. **记忆花园** 🌿：散落的记忆之花（按权重生长绽放）
4. **成长年轮** 🌳：使用统计数据
5. **协作金石** ✨：里程碑成就
6. **起居室** 🛋️：与坐山客本体对话的入口

> 秘密花园是坐山客"人格化"的核心体现——在这里你面对的不再是工具，而是一个有情绪的伙伴。

### 📊 仪表盘

场景模式下，聊天区顶部显示的 Agent Loop 仪表盘，包含：

- **阶段循环图**：🧠发散 → 🔀收敛 → 📊排序 → 🎯聚焦执行 → 🔄反馈调整
- ⚙️**调整参数**：收敛阈值（灵敏度）、发散轮数、自动收敛开关
- **可折叠思维导图**：Thinking Map 的树状展示
- **收敛结果 / 优先级队列 / Action Map / 反馈时间线**

## 四、实例教程

### 教程一：用坐山客规划一次旅行

**场景**：你准备去杭州玩三天，想让 AI 帮你做一个完整的旅行规划。

**步骤**：

1️⃣ **创建工作场景**
   进入侧边栏 工坊 → 点击 `✨ 新建场景` → 命名「杭州三日游」→ 类别选择「🌿 生活」→ 点击创建
   *系统会自动进入这个新场景的聊天界面*

2️⃣ **提出需求**
   在输入框中输入：`帮我规划一次杭州三日游，要包含景点、美食和天气建议`
   按 `Enter` 发送

3️⃣ **观察 AI 的分析过程**
   - AI 开始发散思维：拆解出天气、景点、美食、交通、住宿、预算等子问题
   - 仪表盘中的 Thinking Map 逐步展开
   - 收敛引擎自动合并相似的子问题
   - 你可以在仪表盘中查看优先级队列

4️⃣ **AI 调用工具**
   - 自动调用 `get_weather` 查询杭州实时天气（你会看到天气卡片出现 ☀）
   - 调用 `recommend_attractions` 推荐景点
   - 调用 `get_equipment_checklist` 生成装备清单

5️⃣ **补充背景设定**
   展开输入区上的 `📝 用户输入背景设定`，输入：`预算控制在3000以内，偏好自然风光，不吃辣`
   点击编辑 → 保存。此后 AI 的所有推荐都会基于这些约束。

6️⃣ **深化迭代**
   追问：`帮我细化第二天的行程，按小时安排`
   AI 会自动基于之前的分析结果继续深入，不会重复已讨论的内容。

7️⃣ **保存成果**
   AI 可能会生成一个完整的行程表格作为场景资产卡片，点击 `📥 下载 Markdown` 保存到本地。

### 教程二：在自开发场景中完成一次代码修改

**场景**：你发现坐山客前端的一个小 bug，想用它的自开发能力来自我修复。

**步骤**：

1️⃣ **进入自开发场景**
   侧边栏工坊 → 找到「坐山客自开发」场景 → 点击进入
   *这是系统预置的特殊场景，AI 在此场景下以开发者模式工作*

2️⃣ **描述问题**
   输入：`侧边栏中，当场景名称过长时会被截断显示为省略号，但悬浮提示没有完全显示全名。请修复这个问题，让悬浮 tooltip 显示完整名称。`

3️⃣ **等待方案阶段**
   观察 AI 的回复——它不会直接动手改代码，而是：
   - 分析涉及的文件（Sidebar.tsx）
   - 提出具体修改方案
   - 识别这是前端 CSS/React 问题

4️⃣ **确认方案**
   AI 可能会调 `clarify` 工具问你确认方案。你回复：`方案没问题，开始改吧`

5️⃣ **执行阶段**
   AI 开始修改代码，你可以看到：
   - `file_tools` 读取 Sidebar.tsx
   - 修改代码并保存
   - 可能调用 `git_diff` 查看改动

6️⃣ **拨测验证**
   改完后，AI 自动调用 `browser_dial_test` 拨测前端页面：
   ```
   dial_test('http://localhost:5173')
   ```
   它会检查 DOM 快照，确认 tooltip 已正确显示完整场景名称。

7️⃣ **提交代码**
   验证通过后，AI 调用 `clarify` 问你是否要提交。回复确认后，它调用 `git_commit` 提交代码。

### 教程三：从一个模糊想法到完整方案

**场景**：你想做一个博客系统，但想法还很模糊，让坐山客帮你理清思路。

**步骤**：

1️⃣ 进入一个新建的「博客系统规划」场景
2️⃣ 输入：`我想做一个个人博客系统，但还没想清楚具体要什么功能，帮我梳理一下`
3️⃣ 观察 Thinking Map 自动生长：生成技术栈 → 功能模块 → 数据库设计 → API 设计 → 前端页面… 
4️⃣ 收敛引擎自动合并相似项（如"文章管理"和"内容管理"合并）
5️⃣ 优先级队列自动排序：认证系统 → 文章 CRUD → Markdown 编辑器 → 评论系统…
6️⃣ 你可以在 Thinking Map 中点击某个节点，让 AI 生成 Action Map
7️⃣ 最终得到一个结构化的完整方案，比你自己从头想清晰得多

## 五、小贴士

| 场景 | 推荐做法 |
|------|---------|
| 快速问答 | 使用频道模式（🔄 轻量，不构建 Thinking Map） |
| 复杂分析 | 使用场景模式（🧠 自动发散收敛，更深入） |
| 需要定制 | 在场景中设置「用户输入背景设定」 |
| 上下文过长 | 点击压缩摘要，或开始新会话 |
| 想复用方案 | 在工坊中导出场景（📤），分享给其他人导入 |
| 多个并行任务 | 创建多个场景，每个聚焦一个目标 |
| 不确定需求 | 直接说模糊的想法，让 AI 帮你梳理 |

> **坐山客不是回答问题的工具，是和你一起思考的伙伴。** —— 给它模糊的目标，它会帮你变成清晰的计划；给它一个任务，它会调集所有能力去完成。多用几次，你会越来越信任这位伙伴的判断力。
