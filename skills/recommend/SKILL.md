# Recommend — 天气推荐引擎

## 概述

「天气 → 场景 → 景点」智能推荐管道。输入城市名，获取实时天气，分类判断，筛选匹配景点，按置信度排序输出。

## 核心流程

1. **获取天气** — 调用天气接口获取实时天气
2. **天气分类** — 根据天气描述、温度、风速、湿度归入预定义分类
3. **场景映射** — 按分类查映射表，获推荐场景类型
4. **景点筛选** — 筛选满足天气分类和温度区间的景点
5. **排序输出** — 按置信度综合排序

## 天气分类体系

| 分类 | 优先级 | 条件 | 默认场景 |
|------|--------|------|----------|
| `extreme` | 7 | 台风、沙尘暴、冰雹、暴雪、暴雨 | 室内避险 |
| `snowy` | 6 | 雪，< 5°C | 户内 |
| `rainy` | 5 | 雨，< 32°C | 户内 |
| `hot` | 4 | 晴/多云，≥ 35°C | 户内避暑 |
| `cold` | 3 | ≤ 0°C | 户内保暖 |
| `windy` | 2 | 大风 > 8级 | 户内 |
| `sunny` | 1 | 晴/多云，10~30°C | 户外 |
| `overcast` | 1 | 阴，10~28°C | 户外 |

优先级规则：按优先级降序判定，匹配即停。

## 返回格式

```python
{
    "city": str,           # 城市名
    "weather": {...},      # 天气数据
    "category": str,       # 天气分类标签
    "scene": str,          # 推荐场景类型
    "confidence": float,   # 分类置信度 0.0 ~ 1.0
    "items": [{            # 推荐景点列表，按置信度降序
        "name": str,
        "category": "户外" / "户内",
        "tags": [str],
        "address": str,
        "note": str,
        "score": float,
        "reason": str
    }]
}
```

## 使用场景

- 根据天气推荐当日活动
- 嵌入自动化决策（如晨报推送出行建议）
- 条件判断后的精细化推荐

## 注意事项

1. **城市限制：** 当前主要覆盖天津亲子景点数据集
2. **天气回退：** 无法获取实时数据时使用静态回退，推荐结果可能不准确
3. **分类优先级：** 高优先级覆盖低优先级
4. **亲子导向：** 当前景点数据以亲子场景为主
5. **无实时开放状态：** 景点是否开放需额外确认
